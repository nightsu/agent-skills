# 分类与映射规则

## 节点分类

- `api_bound`：应该来自接口的业务数据
- `ui_static`：容器、装饰、图标、固定结构、交互壳层
- `ui_copy`：用户可见但不属于业务数据的文案
- `unknown`：无法稳定判断，需要确认

## `api_bound` 的强信号

- 列表、卡片、表格、评论流、网格中的重复内容
- 用户资料、商品、订单、指标、时间、状态、地址
- 代表远程资源、上传内容或实体媒体的图片
- 会随着底层数据变化而变化的字段

## `ui_static` 的强信号

- 状态栏、顶部壳层、导航框架、设备外壳、分割线
- 图标、按钮、容器、间距、阴影、装饰
- 本身不承载业务意义的布局包装层

## `ui_copy` 的强信号

- 空状态文案
- 提示文案、辅助文案、占位式文案
- CTA 文案、标签、说明性字符串

## 确认策略

- 低置信度映射要确认
- 来源不明确的图片要确认
- 可能是复制内容也可能是绑定数据的长文本要确认
- 只有模式不一致的重复组件才需要确认

## 映射输出字段

- `node_id`
- `binding_type`
- `ui_role`
- `api_field`
- `confidence`
- `needs_user_confirmation`

## 接口文档时机

- 先把页面降到业务层，再读接口文档
- 只拉取能解决 `api_bound` 或 `unknown` 的字段
- 不要因为有文档就把每一层都查一遍
