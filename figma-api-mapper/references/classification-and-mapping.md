# 分类与映射规则

这个参考文档用于补充 `SKILL.md` 中的分类规则，重点放在边界场景和重复组件判断，不重复主流程。

## 节点分类

- `api_bound`：应该来自接口的业务数据
- `ui_static`：容器、装饰、图标、固定结构、交互壳层
- `ui_copy`：用户可见但不属于业务数据的文案
- `unknown`：无法稳定判断，需要确认

## `api_bound` 的强信号

- 列表、卡片、表格、评论流、网格中的重复内容
- 用户资料、商品、订单、指标、时间、状态、地址
- 代表远程资源、上传内容或实体媒体的图片
- 会随着底层数据变化而变化的字段
- 同一组件实例之间主要变化的是文本、图片、数字或状态

## `ui_static` 的强信号

- 状态栏、顶部壳层、导航框架、设备外壳、分割线
- 图标、按钮、容器、间距、阴影、装饰
- 本身不承载业务意义的布局包装层
- 交互入口本身固定，但其角标、数量、状态文案可能另算

## `ui_copy` 的强信号

- 空状态文案
- 提示文案、辅助文案、占位式文案
- CTA 文案、标签、说明性字符串
- 在多个页面复用且不随实体数据变化的文案

## 常见边界场景

### 标签和角标

- “限时优惠”“热门”“推荐”这类标签，如果没有证据表明来自接口，先判为 `ui_copy`
- 如果同一标签在不同卡片之间会变化，或接口文档里存在对应字段，可提升为 `api_bound`

### 按钮与按钮文案

- 按钮容器、描边、背景、图标通常是 `ui_static`
- 按钮上的“立即购买”“查看更多”通常是 `ui_copy`
- 如果按钮文案会随状态切换，例如“已报名 / 去报名”，则文案本身可能是 `api_bound`

### 图片

- 商品图、头像、课程封面、文章头图优先视为 `api_bound`
- 背景纹理、装饰插画、品牌固定视觉优先视为 `ui_static`
- 截图无法判断图片来源时，标为 `unknown`

### 长文本

- 商品简介、评论正文、动态内容优先视为 `api_bound`
- 固定说明、协议摘要、引导提示优先视为 `ui_copy`
- 如果一句长文本既像运营文案又可能是后台配置，先标 `unknown`

## 重复组件策略

- 发现重复卡片、列表项、表格行时，先抽象模板，再提取模板中的可变字段
- 同一模板默认只映射一次，并用 `repeat_group`、`list_item` 或类似字段标记其属于重复单元
- 只有当某个实例比其他实例多出独有结构，才单独列出额外节点

## 确认策略

- `confidence < 0.70` 的映射要确认
- 来源不明确的图片要确认
- 可能是固定文案也可能是绑定数据的长文本要确认
- 视觉分析和结构化节点结论冲突时要确认
- 只有模式不一致的重复组件才需要确认

## 映射输出字段

- `node_id`
- `binding_type`
- `ui_role`
- `api_field`
- `confidence`
- `needs_user_confirmation`
- `reason`
- `repeat_group`

## 接口文档时机

- 先把页面降到业务层，再读接口文档
- 只拉取能解决 `api_bound` 或 `unknown` 的字段
- 不要因为有文档就把每一层都查一遍
- 如果只有接口名称没有字段示例，先给结构草案，再列出待确认字段
