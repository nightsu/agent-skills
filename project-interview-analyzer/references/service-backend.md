# 后端服务 / 数据 / AI 应用参考

当仓库更像后端服务、BFF、数据管道、任务系统、AI 应用或 RAG 服务时，使用这份参考。

## 重点关注

- 服务边界：对外 API、内部模块、上游 / 下游依赖分别是什么。
- 领域模型：核心实体、状态流转、权限边界和一致性要求。
- 数据链路：数据库、缓存、队列、对象存储、搜索索引、向量库或模型调用如何协作。
- 异步任务：任务创建、重试、幂等、超时、补偿和失败告警。
- 可靠性：错误处理、限流、降级、可观测性、审计日志和回滚策略。
- 安全与合规：认证、授权、敏感信息处理、租户隔离和数据权限。

## 值得讲的实现

- 清晰的领域服务 / repository / adapter 分层。
- 复杂状态机、审批流、结算流、导入导出或批处理任务。
- 幂等提交、重试补偿、事务边界和最终一致性设计。
- BFF 聚合、协议适配、第三方系统集成。
- RAG 检索链路、prompt / tool 编排、模型降级、评测与可观测性。

## 特别留意

- 不要把“有接口”当成亮点；亮点要能说明边界、状态、失败场景或复用价值。
- 如果涉及 AI 能力，要区分事实能力、推测用途和未验证效果。
- 如果没有压测、指标或线上证据，不要编造性能提升、准确率提升或成本下降。

## 面试讲法

- 先讲业务约束和系统边界，再讲技术实现。
- 重点说清输入、处理、输出、副作用和失败兜底。
- 对指标类价值保持保守：可以说“设计目标是降低延迟 / 提升稳定性”，但不要写成已达成结果，除非有证据。
