# 基建参考

当仓库更像基建、平台、工具链、SDK、CLI、脚手架或研发效率平台时，使用这份参考。

## 重点关注

- 这套能力服务谁，解决哪类研发问题。
- 它提升的是效率、稳定性、规范性，还是可观测性。
- 系统边界在哪里：构建、发布、测试、代码生成、监控、平台集成。
- 是否有明显的平台化设计或组织级复用。

## 值得讲的实现

- 构建 / 发布 / 测试自动化
- 脚手架、模板系统、代码生成
- SDK / CLI / 平台接入层
- 可观测性：日志、埋点、告警、链路跟踪
- 规范治理：lint、类型、质量门禁、流水线

## 特别留意

- 不要只说“做了 CI/CD”或“做了脚手架”。
- 要讲：
  - 目标用户是谁
  - 旧流程有什么问题
  - 新方案如何降低成本或减少错误
  - 维护成本和 tradeoff 是什么

## 面试讲法

- 更强调平台收益、组织复用和稳定性。
- 尽量量化影响：时间、步骤、出错率、回归成本、覆盖范围。
