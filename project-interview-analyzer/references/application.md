# 业务应用参考

当仓库的核心是页面、路由、服务层和用户流程时，使用这份参考。

## 重点关注

- 页面结构如何组织：按业务域、按功能域，还是按技术层。
- 路由、状态、服务层的边界是否清晰。
- 用户完成关键任务的路径是什么。
- 哪些页面或流程最复杂，复杂度来自哪里。
- 是否存在跨系统协作，例如登录、支付、文件、审批、消息、数据同步。

## 值得讲的实现

- 多步骤编辑器 / 向导页
- 嵌套表单 / 可编辑表格 / 树结构编辑
- 复杂列表页：筛选、分页、展开行、批量操作、导出
- 预览驱动或试算驱动的提交流程
- 文件上传 / 下载 / 异步任务链路
- 并发保护、误操作保护、一致性保护

## 特别留意

- 页面是单纯展示，还是承载状态治理。
- 复杂度是否被收敛在抽象层，还是散落在页面中。
- 是否存在 create / modify / copy / preview 多态共存。
- 是否存在前后端结构不一致，需要做适配或转换。

## 面试讲法

- 不要只说“做了很多页面”。
- 要把重点放在：
  - 复杂业务建模
  - 状态治理
  - 交互设计
  - 风险控制
  - tradeoff
