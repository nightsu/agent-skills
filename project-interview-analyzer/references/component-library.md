# 组件库参考

当仓库更像组件库、设计系统、UI 基础能力或前端基础包时，使用这份参考。

## 重点关注

- 组件或能力的抽象边界是什么。
- API 设计是否稳定、一致、可组合。
- 是否支持主题、样式覆盖、插槽、受控 / 非受控模式。
- 是否有文档、示例、stories、测试矩阵、兼容策略。
- 是否有可访问性、性能、浏览器兼容或跨框架适配要求。

## 值得讲的实现

- 高复用组件的 API 设计
- 主题系统 / tokens / 变量体系
- 复杂交互组件，例如表格、树、表单控件、拖拽组件
- 受控 / 非受控、同步 / 异步状态的统一抽象
- 向后兼容、迁移策略、breaking change 管理

## 特别留意

- 不要把“组件数量多”当成亮点。
- 真正值得讲的是：
  - 抽象是否合理
  - API 是否可扩展
  - 不同业务是否能稳定复用
  - 复杂交互在组件层是如何收口的

## 面试讲法

- 重点讲“抽象边界”和“约束设计”。
- 多谈为什么这样定义 API，少谈页面业务。
