# §4b — `figma-design-token` 契约(新建)

> 上一篇:[04a UI-API Mapper](./04a-ui-api-mapper.md) · 下一篇:[04c Emit Spec](./04c-emit-spec.md)

---

## 职责

从指定 Figma node 抽取**视觉级 token**(尺寸 / 字体 / 字重 / 色值 / 间距 / 圆角 / 资源引用),
输出 `design-token-patch.md`。

**只补充精确视觉信息**,不承担语义推断,不做节点分类(那是 §4a 的事)。

---

## 前置条件

- `component-mapping.md` 已存在且非占位(知道有哪些业务模块需要 token)
- 用户提供 Figma file key + node id(可以复用上一阶段的;orchestrator 在内存里
  缓存上一次 figma_file_key/node_id,在本阶段交互时作为默认值提示用户复用)

|---|---|---|
| `figma_file_key` | ✅ | 可复用 §4a 的值 |
| `figma_node_id` | ✅ | 可复用 §4a 的值 |

| 文件 | 用途 |
|---|---|
| `docs/design/<feature>/component-mapping.md` | 知道有哪些模块,只对模块抽 token |

---

## 调用外部能力

- **Figma MCP** —
  - `get_variable_defs`:获取 Figma variables(色值、间距 token 等)
  - `get_design_context`:获取节点的视觉细节(尺寸、字体、padding 等)
- 严格只读传入 node 及其子节点,不扩散

## 产物

| 路径 | 类型 | 说明 |

| `docs/design/<feature>/inputs.md` | 追加一条 | 含 figma_file_key, node_id, exported assets references, token_version |

## 产物格式(`design-token-patch.md`)

```markdown
# Design Token Patch — <feature>

> Generated by figma-design-token@0.1.0 at <ISO8601>
> Source: figma file=<file_key> node=<node_id> (<node_name>)

## Page Metrics
| Item | Value |
|---|---|
| Design width | 375px |
| Page padding | 12px |
| Section gap | 16px |

## Module Tokens

### DiamondPreviewCard
| Token | Value | Source |
|---|---|---|
| borderRadius | 12px | direct |
| padding | 12px 16px | direct |
| background | #FFFFFF | direct |
| titleFontSize | 14px | direct |
| valueFontSize | 24px | direct |
| valueFontWeight | Semi Bold | direct |
| valueColor | #1A1A1A | variable: color/brand/primary |

### OperationEntryGrid
| Token | Value | Source |
|---|---|---|
| columnCount | 4 | INFERRED (from 4-instance grid) |
| itemGap | 8px | direct |
| iconSize | 32px | direct |
| labelFontSize | 12px | direct |

## Variables (from Figma variables)
| Name | Value | Used by |
|---|---|---|
| color/brand/primary | #FF6A3D | DiamondPreviewCard.valueColor |
| spacing/card/padding | 12px | DiamondPreviewCard.padding |

## Assets
| Asset | Node ID | Usage | Suggested Export |
|---|---|---|---|
| banner_image | 123:789 | BannerCarousel | png |
| icon_diamond | 123:790 | DiamondPreviewCard | svg |
| icon_arrow | 123:791 | DiamondPreviewCard | svg |
| bg_card | 123:792 | DiamondPreviewCard.background | png (or skip if pure color) |

## Ignored
- StatusBar / HomeIndicator / 装饰背景(已在 component-mapping 阶段过滤)
```

**格式约定:**
- 每个 module 与 `component-mapping.md` 的 module 名一一对应
- `Source` 列三种取值:
  - `direct` — 直接从 Figma 节点读出
  - `variable: <name>` — 关联到一个 Figma variable
  - `INFERRED (...)` — 推断而非 Figma 直接给出(需在 self-check 中爆警告)
- assets 只输出**引用**(node id + 推荐格式),不下载

---

## 关键行为

1. 严格只读传入 node 及其子节点,**不扩散**
2. 对 `component-mapping.md` 中标记为 `api_bound` 的节点,**只抽取 token,不抽取文本内容**
   (防止把 Figma 里的样例数据当成业务事实)
3. 抽取 Figma variables 时,关联到使用它的模块
4. assets 只输出引用(node id + 推荐格式),**不下载**
5. 处理字体名称的常见陷阱:`Inter` 的字重是 `"Semi Bold"`(有空格),不是 `"SemiBold"`

---

## 不做的事

- ❌ 不做节点分类(那是 §4a)
- ❌ 不下载图片资源(只给引用,实际下载交给用户/后续 skill 在第 2/3 版处理)
- ❌ 不读取整个文件,不读兄弟 frame
- ❌ 不抽取 `ui_copy` 节点的文本内容(那些应该用户来约定)
- ❌ 不修改 `component-mapping.md`

---

## 错误处理

| 情况 | 处理 |
|---|---|
| Figma MCP 调用失败 | 透传错误,产物**不落盘** |
| node id 在文件中找不到 | 报错并提示用户确认 URL 是否正确 |
| component-mapping 中存在的 module 在 Figma 节点中找不到对应子节点 | 在 `## Module Tokens` 段列空表 + 写入 self-check 警告;不阻塞产物落盘 |
| Figma variable 解析失败 | 该 token 标 `Source: direct (variable resolve failed)`,值仍写入 |

---

## 自查规则(用于 §5 Review Gate)

| 项 | 阈值 | 提示文案 |
|---|---|---|
| `Source: INFERRED` token 数 | > 0 | "N 个 token 标记为 INFERRED,建议人工核对" |
| `component-mapping.md` 出现但本产物未抽 token 的 module 数 | > 0 | "N 个 module 在 component-mapping 出现但本产物缺 token: [...]" |
| Figma variable 数 | > 0 (仅 ℹ 信息) | "N 个 Figma variable 被关联" |
