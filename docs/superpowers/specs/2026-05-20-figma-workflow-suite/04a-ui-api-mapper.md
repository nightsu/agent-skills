# §4a — `figma-ui-api-mapper` 契约(改名自 `figma-api-mapper`)

> 上一篇:[03 Orchestrator](./03-orchestrator.md) · 下一篇:[04b Design Token](./04b-design-token.md)

---

## 职责

把 Figma node 的节点树和接口字段做语义化映射,输出节点分类 + UI↔API 字段对照。

**职责保持现状**(节点 4 分类 + 字段映射),只是改名 + 重定位到阶段 C 末尾。

**4 分类沿用现有 mapper:**
- `api_bound` — 应该来自接口的业务数据
- `ui_static` — 容器、装饰、图标、固定结构、交互壳层
- `ui_copy` — 用户可见但不属于业务数据的文案
- `unknown` — 无法稳定判断,需要确认

---

## 前置条件

- `clarified-requirement.md` 存在且非占位
- `ui-understanding.md` 存在且非占位
- `api-mapping.md` 存在且非占位
- 用户提供 Figma file key + node id(本阶段输入)

---

## 输入(本阶段收集)

| 字段 | 必传 | 说明 |
|---|---|---|
| `figma_file_key` | ✅ | Figma file key,从 URL 截取 |
| `figma_node_id` | ✅ | 起始 node id,形如 `123:456` |
| `max_depth` | ⬜ | 节点遍历最大深度,默认 6 |
| `exclude_patterns` | ⬜ | 节点名忽略正则列表;默认沿用现有 mapper 的 IGNORE_NAME_PATTERNS |

---

## 读取(上游产物)

| 文件 | 用途 |
|---|---|
| `docs/design/<feature>/clarified-requirement.md` | 理解业务语义(用户角色、scope、状态) |
| `docs/design/<feature>/ui-understanding.md` | 对齐推测组件(模块树、可疑组件) |
| `docs/design/<feature>/api-mapping.md` | 字段匹配的字典 |

---

## 调用外部能力

- **Figma MCP** — `get_metadata` 获取节点结构,`get_design_context` 获取节点细节
- **严格只读传入的 node**,不读取兄弟 frame,不扩散到整个文件

---

## 产物

| 路径 | 类型 | 说明 |
|---|---|---|
| `docs/design/<feature>/component-mapping.md` | 主产物 | Markdown 表格 |
| `docs/design/<feature>/inputs.md` | 追加一条 | 含 figma_file_key, node_id, node_name, mapper_version |

---

## 产物格式(`component-mapping.md`)

```markdown
# Component Mapping — <feature>

> Generated by figma-ui-api-mapper@0.4.0 at <ISO8601>
> Source: figma file=<file_key> node=<node_id> (<node_name>)

## Summary

- api_bound: 12
- ui_static: 8
- ui_copy: 5
- unknown: 2
- needs_confirmation: 4

## Modules

### DiamondPreviewCard
| node_id | binding_type | ui_role | api_field | confidence | needs_confirmation | reason |
|---|---|---|---|---|---|---|
| diamond_value | api_bound | primary_value | user.diamond.balance | 0.96 | false | — |
| diamond_label | ui_copy | label | — | 0.92 | false | — |
| record_entry_icon | ui_static | entry_icon | — | 0.95 | false | — |
| record_entry_text | ui_copy | entry_text | — | 0.88 | false | — |

### OperationEntryGrid
| node_id | binding_type | ui_role | api_field | confidence | needs_confirmation | reason | repeat_group |
|---|---|---|---|---|---|---|---|
| entry_icon | api_bound | list_item_icon | entries[].iconUrl | 0.94 | false | — | entry_grid_item |
| entry_label | api_bound | list_item_label | entries[].title | 0.95 | false | — | entry_grid_item |
| entry_badge | unknown | list_item_badge | — | 0.55 | true | 可能为运营标签 / 可能为固定 | entry_grid_item |

## Open Questions
- node `cover_image_123`: 来源不明(可能为 banner 接口 / 可能为固定素材)
- node `tag_456`: 标签文案可能来自接口 entries[].tag

## Ignored Nodes
- StatusBar / HomeIndicator / DeviceFrame / 装饰背景
```

**格式约定:**
- 每个 module 一个 `###` 子节,模块名来自 `ui-understanding.md` 的"Suspected Components"或推断
- 重复列表项只一行,标 `repeat_group`(同现有 mapper)
- `confidence < 0.7` 自动标 `needs_confirmation: true`(同现有 mapper)
- `unknown` 节点必须出现在 "Open Questions" 段
- "Ignored Nodes" 段汇总被过滤的节点类型(不逐个列)

---

## 关键行为(沿用现有 mapper 的判定规则)

1. 严格只读传入的 node,不读取兄弟 frame,不读取整个文件
2. 重复列表项**只映射一次模板**,标 `repeat_group`
3. 置信度 < 0.7 自动标 `needs_confirmation: true`
4. 与现有 mapper 的判定规则、置信度阈值、ignored patterns **完全保持**
5. 不强行猜测图片内容
6. 不展开重复列表

---

## 不做的事

- ❌ 不输出 design token(尺寸/字体/色值)→ 那是 [§4b](./04b-design-token.md) 的事
- ❌ 不读取整个 Figma 文件
- ❌ 不强行猜测图片内容
- ❌ 不展开重复列表
- ❌ 不下载资源(只在表格中标注 ui_role,资源引用由 §4b 处理)

---

## 错误处理

| 情况 | 处理 |
|---|---|
| Figma MCP 调用失败 | 透传错误,产物**不落盘** |
| node id 在文件中找不到 | 报错并提示用户确认 URL 是否正确 |
| 上游产物缺失 | orchestrator 已拦截,本 skill 不会被调用 |
| 上游产物存在但内容矛盾(如 api-mapping 缺字段) | 写入 "Open Questions" 段,不阻塞产物落盘 |

---

## 自查规则(用于 §5 Review Gate)

| 项 | 阈值 | 提示文案 |
|---|---|---|
| `needs_confirmation: true` 节点数 | > 0 | "N 个节点标记需用户确认" |
| `unknown` 节点数 | > 0 | "N 个节点分类为 unknown,见 Open Questions" |
| 低置信度数(<0.7) | > 0 | "N 个映射置信度 < 0.7" |
| Open Questions 数 | > 0 | "N 个 open question 待回答,建议先回答再进 phase D" |
