# Markdown 格式化工具

对 Markdown 文件进行标准化格式清理，消除导出工具或手写引入的格式问题。

## 前置条件

1. **获取目标文件**：确认用户要格式化的 Markdown 文件路径。接受以下输入方式：
   - 用户提供文件路径
   - 其他技能传入文件路径
   - 用户粘贴 Markdown 文本（此时先写入临时文件再处理）

2. **确认格式化范围**：默认执行全部格式化规则。若用户只需部分规则（如“只清理 HTML 标签”），记录限定范围。

## 工作流程

### 阶段一：格式分析

1. 读取目标 Markdown 文件全文。
2. 识别存在的格式问题，统计各类问题数量：
   - 标题层级跳跃（如 H1 直接到 H3）
   - 连续空行过多（3 行及以上）
   - HTML 残留标签（`<p>`、`<br/>`、`<span>`、`<div>`、`<ac:*>` 等）
   - 分隔线格式不统一（`***`、`___` 等）
   - 链接格式有多余空格
   - 表格分隔行不规范
3. 向用户报告发现的问题概要。

### 阶段二：执行格式化

使用本技能目录下的 `scripts/markdown-lint.sh` 脚本执行格式化：

```bash
bash <skill-dir>/scripts/markdown-lint.sh <target-file>
```

脚本执行以下规则：
- **折叠空行**：3+ 连续空行 -> 2 行
- **清除行尾空白**
- **清理 HTML 残留**：移除空 `<p>`、`<br/>`、空 `<div>`、`<span>` 包裹、`<ac:*>` 残留
- **统一分隔线**：`***`、`___`、松散 `- - -` -> `---`
- **规范链接格式**：`[ text ]( url )` -> `[text](url)`
- **修复表格分隔行**：确保每列至少 3 个短横线

### 阶段三：验证与报告

1. 重新读取格式化后的文件，确认格式化已生效。
2. 向用户报告：
   - 处理了哪些类型的格式问题
   - 修改前后的行数变化
   - 若被其他技能调用，直接返回处理后的文件路径

## 缺失信息处理

- 若文件不存在或路径无效，提示用户检查路径。
- 若文件内容不是 Markdown（如二进制文件），拒绝处理并告知用户。
- 脚本执行失败时，报告错误信息，不修改原文件。

## 参考文件
