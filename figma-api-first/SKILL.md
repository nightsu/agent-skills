# Figma API First

## Position in figma-workflow

本 skill 是 `figma-workflow-suite` 的 Phase C1:

```text
phase A → phase B → phase C1 → phase C2 → phase D → phase E
clarified-requirement.md → ui-understanding.md → api-mapping.md → component-mapping.md → design-token-patch.md → implementation-spec.md
```

`figma-api-first` 产出 `api-mapping.md`。
`figma-ui-api-mapper` 消费 `api-mapping.md` 并结合 Figma node 产出 `component-mapping.md`。

## Prerequisites

- `feature=<feature-name>`
- `docs/design/<feature>/clarified-requirement.md` 存在且非占位
- `docs/design/<feature>/ui-understanding.md` 存在且非占位
- 用户粘贴接口结构,支持 TypeScript type / JSON response / 字段清单 / 接口说明文本

## Calling convention

```text
figma-api-first feature=<feature-name>
```

`feature=` 必传。调用后,用户可以在对话中粘贴接口返回类型、JSON 示例、字段清单或接口说明文本。

## 适用场景

- 已完成 Phase A / B,准备进入 C1
- 需要把接口返回结构先整理成稳定字段事实
- 希望让 `figma-ui-api-mapper` 后续基于同一份 `api-mapping.md` 做 UI/API 绑定
- 用户只有粘贴的接口结构,暂时没有可自动抓取的平台入口

## 目标

本 skill 只做接口事实整理:

- 输出文件固定为 `docs/design/<feature>/api-mapping.md`
- 向 `docs/design/<feature>/inputs.md` 追加一条 audit 记录
- 将 API facts 与 UI binding guesses 分开处理
- 借助 A/B 产物中的 module 信息生成 UI Field 候选,但不伪造接口字段
- 将不确定字段、状态、分页、权限、兜底策略写入 `Open Questions`

## 不承担

- 不接 YApi / Swagger / OpenAPI 自动抓取
- 不调用 Figma MCP
- 不读取 raw Figma JSON
- 不产出 `component-mapping.md`
- 不做 UI 节点绑定
- 不修改业务代码
- 不生成 OpenSpec proposal

## 工作流

1. **解析 feature**
   - 缺失 `feature=<feature-name>` 时拒绝执行,提示用法

2. **确认目录**
   - 确保 `docs/design/<feature>/` 存在
   - 如果目录刚创建,打印路径并要求用户确认 feature 拼写后再继续

3. **读取上游产物**
   - 读取 `docs/design/<feature>/clarified-requirement.md`
   - 读取 `docs/design/<feature>/ui-understanding.md`
   - 任一缺失或仍是模板占位时拒绝执行

4. **获取接口结构**
   - 如果用户尚未提供接口结构,要求用户粘贴 TypeScript type / JSON response / 字段清单 / 接口说明文本

5. **抽取 API facts**
   - API name 或 URL
   - Method
   - response root shape
   - 字段完整路径
   - 字段类型
   - loading / empty / error 状态线索

6. **对齐 A/B module**
   - 只把 A/B 中的 module 当作 UI Field 候选
   - 不从 UI 槽位伪造 API 字段
   - 不假设每个 API 字段都会展示到 UI

7. **生成 `api-mapping.md`**
   - 按 [references/api-mapping-template.md](references/api-mapping-template.md) 填充
   - 保持 Markdown 表格格式,便于 review 和 git diff

8. **记录不确定项**
   - 低置信度映射
   - 缺失类型
   - 分页、权限、错误码、空态、兜底展示问题
   - 全部写入 `Open Questions`

9. **追加 audit 记录**
   - 在 `docs/design/<feature>/inputs.md` 追加一条记录
   - 字段包括 source type、timestamp、api_first_version、field_count、open_questions_count

10. **输出 self-check**
    - 打印自查结果,供 `figma-workflow` review gate 展示

## 输出结构

```markdown
# API Mapping — <feature>

> Generated by figma-api-first@0.1.0 at <ISO8601>
> Source: pasted schema

## Data Sources
| UI Module | API | Method | Description |
|---|---|---|---|
| <module or page> | <api name/url> | GET/POST/UNKNOWN | <business purpose> |

## Field Mapping
| UI Field | API Field | Type | Transform |
|---|---|---|---|
| <ui candidate or UNKNOWN> | data.foo.bar | number | percent / duration / currency / count / raw |

## State Mapping
| State | Trigger | UI Behavior |
|---|---|---|
| loading | request pending | show loading placeholder |
| empty | response success but key list/count is empty | show empty state |
| error | request failed or success=false/code!=0 | show error state |

## Open Questions
- [ ] <unresolved api / type / pagination / permission / fallback question>
```

## 填充规则

- API Field 使用完整 dotted path,例如 `data.firstContactSection.callOutRate.percent`
- Type 必须来自接口事实;无法确认时写 `unknown`
- Transform 只描述展示转换,例如 `percent` / `duration` / `currency` / `count` / `raw`
- UI Field 可以使用 A/B 的 module hints;不稳定时写 `UNKNOWN`
- 不从 UI slot 发明 API 字段
- 不假设每个 API 字段都会展示

## Self-check

| Check | Warning |
|---|---|
| `Data Sources` empty | "Data Sources 缺失,不能进入 C2" |
| `Field Mapping` empty | "Field Mapping 缺失,不能进入 C2" |
| field type contains `unknown` | "存在 unknown 类型,建议 review 后再继续" |
| `Open Questions` contains non-deferred items | "存在未解决接口问题,建议 review 后再继续" |
| template placeholder remains | "产物仍含模板占位,orchestrator 会视为未完成" |

self-check 不阻塞产物落盘;是否继续由 `figma-workflow` 的 review gate 决定。

## 下游衔接

- C1 完成后回到 `figma-workflow` review gate,不自动进入 C2
- 进入 C2 时,`figma-ui-api-mapper` 读取本 skill 生成的 `api-mapping.md`
- 后续 Phase D / E 不需要知道 `api-mapping.md` 来自手填还是 `figma-api-first`

## 不要做的事

- ❌ 不接 YApi / Swagger / OpenAPI 自动抓取
- ❌ 不调用 Figma MCP
- ❌ 不读取 raw Figma JSON
- ❌ 不从 UI 猜接口字段
- ❌ 不产出 `component-mapping.md`
- ❌ 不修改业务代码
- ❌ 不生成 OpenSpec proposal
